# Contributing Principles

This repository is a public profile workspace. The same standards apply to external open-source contributions.

## Contribution standards

- Keep each change small and focused.
- Explain the problem before describing the fix.
- Include verification steps in the PR description.
- Avoid unrelated formatting or refactors.
- Prefer maintainable changes over clever changes.

## Before opening a PR

- Read the repository README and contribution guide.
- Check recent issues and pull requests for related work.
- Reproduce the problem or verify the documentation gap.
- Make one clear change per commit.
- Add a short commit comment explaining the value of the change.

## PR quality bar

A good PR should answer:

- What problem does this solve?
- Why is this change scoped correctly?
- How was it verified?
- What risk remains?

## Commit format

Use a concise subject and a short body:

```text
Improve setup instructions

This commit clarifies the setup steps so new contributors can run the project without guessing.
```
