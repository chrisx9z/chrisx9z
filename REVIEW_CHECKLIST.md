# Review Checklist

Use this checklist before opening or updating a pull request.

## Scope

- [ ] The PR changes one topic only.
- [ ] The title describes the user-visible improvement.
- [ ] The PR body explains the problem and the solution.
- [ ] No unrelated formatting, renaming, or refactoring is included.

## Verification

- [ ] I checked the changed files locally.
- [ ] I verified links or commands mentioned in documentation.
- [ ] I added or updated tests when behavior changed.
- [ ] I noted any verification that could not be run.

## Maintainer experience

- [ ] The diff is small enough to review quickly.
- [ ] The commit history is readable.
- [ ] Each commit has a short body explaining its value.
- [ ] The PR links the relevant issue when one exists.

## Final check

- [ ] The change is useful even if it is small.
- [ ] The explanation is specific, not generic.
- [ ] Remaining risk is called out clearly.
